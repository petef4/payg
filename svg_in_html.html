Cross browser SVG in HTML

The goal is display an SVG image in any capable browser using valid HTML and SVG.

The following leverages conditional comments, a feature introduced by Microsoft in IE 5. The aim is to embed an SVG image using the non-standard embed element in IE 5, 6, and 7 or the standard object element in Opera, Firefox, etc. The code may need to be updated if IE 8 turns out to support the object element. No JavaScript is used.

It is valid HTML 4.01 Strict.

The size is set to 120px by 40px in both the svg element in the SVG and the object (or embed) element in the HTML.

Circle filled with red and stroked with blue.

Tested on

  • Opera 8.52 to 9.26. Built in SVG support was introduced in O8.
  • Firefox 1.5.0.1. Built in SVG support was introduced in FF1.5.
  • SeaMonkey 1.0. Built in SVG support was introduced in SM1.0.
  • IE 6.0 with ASV 3.03. IE 5 should work too.
  • IE 6.0 and 7.0 with ASV 6.0.
  • Netscape 7.0 with ASV 6.0.
  • Netscape 4.8 with ASV 6.0. The object content is not suppressed.

It may be possible to display using plugins on older browsers but this has not been tested. SVG viewers include ASV3, ASV6, CSV. Older browsers include Netscape 4 and 6, Mozilla to 1.7, Firefox to 1.0, Opera to 7.5. No testing has been done on non-Windows platforms yet.

Validation

The image should look like this:

Try zooming the page to confirm that the vector SVG image is being used. The fallback PNG images become grainier on magnification. FF2 zoom support is lacking, third party extensions may improve that.

Source

The HTML used is

      <!--[if IE]>
      <embed src="circle01.svg" width="120" height="40"
        pluginspage="http://www.adobe.com/svg/viewer/install/">
      <![endif]-->
      <!--[if IE]><![if !IE]><![endif]-->
      <object width="120" height="40" data="circle01.svg"
        type="image/svg+xml"><span>Unable to show SVG. </span>
      </object>
      <!--[if IE]><![endif]><![endif]-->
